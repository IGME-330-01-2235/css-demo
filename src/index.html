css-05 : position:

This branch demonstrates different values of the position property.

All elements on the page (except for the button) start out following static. But clicking the button changes them to follow their declared position.

position: static

The default value of position is "static" - which just means elements follow the Normal Flow.

Setting its top, left, bottom, right, and z-index have no effect.

position: relative

This keeps the original footprint of the element inserted in the Normal Flow.

Setting its top, left, bottom, right, and z-index nudges it from where it would otherwise have been.

position: absolute

This removes the element from the Normal Flow.

Setting its top, left, bottom, right, and z-index nudges it in relation to its first positioned parent.

(since this element has no positioned parent, it gets adopted by the viewport and behaves like fixed.)

This is a remix of absolute, but with a bunch of nested parents with position: relative, to demonstrate the in-relation-to aspect of position: absolute.

Use the inspector to turn off the position property on these parent div elements, and watch how the red box (which is absolute) bounces around.

position: fixed

This removes the element from the Normal Flow.

Setting its top, left, bottom, right, and z-index is like absolute, but in relation to the viewport.

position: sticky

It's a technique for setting a scrolling/sticky header.
